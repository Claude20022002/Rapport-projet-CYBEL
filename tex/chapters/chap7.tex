\subsection{Contributions personnelles}

En tant qu'équipe chargée du projet CYBEL, nos contributions principales
incluent :

\begin{enumerate}
    \item la \textbf{rétro-ingénierie} du protocole de communication du CIOT
    TY1251D, sans documentation constructeur ;
    \item la \textbf{conception et le développement} de l'architecture CYBEL
    (SDK, API, deux interfaces web) ;
    \item la \textbf{résolution du canal TTS} par \texttt{CybelTTSBridge} et
    l'investigation ADB/broadcast ;
    \item le \textbf{déploiement embarqué} Termux et kiosque sur la tablette ;
    \item la \textbf{conception du parcours pédagogique} laboratoire depuis
    \texttt{knowledgeV2-lab.json} ;
    \item la \textbf{documentation technique} (\texttt{docs/}, scripts,
    diagrammes) ;
    \item les \textbf{corrections itératives} terrain (annulation navigation,
    rosbridge, erreurs 604).
\end{enumerate}


\subsection{Difficultés rencontrées et solutions apportées}

Au-delà des difficultés d'investigation (chapitre~5), la phase de
développement et de validation a nécessité des correctifs spécifiques :

\begin{itemize}
    \item \textbf{FastAPI sur Termux} — remplacement par Starlette
    (\texttt{cybel\_lite.py}) ;
    \item \textbf{Écran blanc WebView} — build IIFE et correctifs safe-area ;
    \item \textbf{Saturations \textit{rosbridge}} — \texttt{dev.py} sans
    \texttt{--reload} par défaut, instance unique du backend ;
    \item \textbf{Navigation 604} — affinage progressif des coordonnées sur
    carte SLAM réelle via sessions terrain documentées dans
    \texttt{docs/labo/TERRAIN.md}.
\end{itemize}


\subsection{Limites du projet}

\begin{enumerate}
    \item Dépendance au Wi-Fi du robot — pas de contrôle hors réseau
    \texttt{TY1251D-03195}.
    \item Un seul robot de test — généralisation à d'autres modèles CIOT non
    garantie.
    \item Coordonnées du parcours — calibrage manuel sur carte SLAM.
    \item Pas de ROS natif — dépendance totale à \textit{rosbridge}.
    \item Android~7.1 — contraintes WebView, pas de modules ES modernes.
    \item Backend lite Termux — sous-ensemble des fonctions opérateur.
    \item Pas de CI/CD ni déploiement industrialisé.
    \item Reconnaissance vocale visiteur — non implémentée en v1.
\end{enumerate}

\subsection{Conclusion}

La validation démontre que CYBEL atteint son objectif principal : \textbf{commander
et faire interagir le robot sans l'application propriétaire}. Les tests
unitaires et d'intégration mock sont au vert ; les tests terrain confirment
téléopération, TTS, télémétrie et navigation ponctuelle. La visite guidée
complète sur huit arrêts reste le dernier jalon de validation, conditionnée
par l'affinage des coordonnées et la stabilité réseau. Les limites identifiées
sont connues et documentées ; les perspectives constituent une feuille de
route pour la suite du projet au laboratoire HESTIM.
